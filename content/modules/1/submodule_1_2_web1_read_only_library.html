The Story - 1998, India's First Internet Café

📍 Setting: 1998 - A smoky internet café in Connaught Place, Delhi

You pay ₹60 for 30 minutes. The dial-up modem screams as it connects. You navigate to www.indiatimes.com - one of India's first news websites.

You see: a headline. A photo (loading very slowly, one pixel at a time). A paragraph of text.

You try to comment on the article. Nothing. There is no comment box.

You try to upload your own photo. Nothing. There is no upload button.

You try to rate the article. Nothing. Not possible.

The website is a digital newspaper. You can read it. That's it. You have zero power.

The editor wrote the article. A developer published it. You read it. End of interaction.

Welcome to Web 1.0 - The Read-Only Library.

Now fast-forward to 2024. You open Instagram Reels. In 30 minutes you: scroll through 200 personalized videos, like 40 of them, comment on 3, share 2 to your friends, upload your own reel, get 50 likes, and reply to 10 DMs. You are simultaneously a consumer AND a creator.

The difference between these two experiences - passive reading vs active participation - is the difference between Web 1.0 and Web 2.0. And understanding that difference helps you understand why Web 3.0 (blockchain) matters: because Web 2.0 gave users the power to create, but kept all ownership with the platform. Blockchain gives users power AND ownership.

📺 Perfect Analogy

Web 1.0 = Doordarshan (DD National) in the 1990s. One broadcaster, millions of passive viewers. You watch what they show, when they show it. You cannot interact, upload, or participate. The broadcaster has total control. Now imagine a world where everyone has a TV channel AND a studio - that's Web 2.0. Now imagine that world where you also OWN your channel and can't be shut down - that's Web 3.0.

Background Knowledge & History

3.1 - Tim Berners-Lee Invents the Web (1989 - 1991)

Remember from Sub-Module 1.1: the Internet (TCP/IP infrastructure) existed since 1983. But there was no easy way to share documents across it. If a physicist at CERN wanted to share research with a physicist in Japan, they had to email files - and both needed the same software to open them.

Tim Berners-Lee solved this in 1989 with three inventions that together form the World Wide Web:

InventionWhat It DoesModern Equivalent
HTML (HyperText Markup Language)A standard language to describe how documents look - headings, paragraphs, images, linksThe .html files behind every website
HTTP (HyperText Transfer Protocol)A protocol for requesting and delivering HTML documents over the internetThe "https://" in your browser address bar
URL (Uniform Resource Locator)A unique address for every document on the webwww.google.com, docs.mstblockchain.com
🗝️ Critical Distinction

The Internet is the physical/protocol infrastructure (cables, routers, TCP/IP). The World Wide Web is one application running on top of the internet - a way to share documents. They are not the same thing. Email, WhatsApp video calls, and Zoom also run on the internet but are NOT the web. Web 1.0 refers specifically to the early era of the WWW - roughly 1991 to 2003.

3.2 - The Era of Static Pages (1991 - 2003)

The early web had only one mode: a developer writes HTML → uploads the file to a server → users fetch and read it. The same file. To everyone. Forever. Until the developer manually updates it.

3.3 - GeoCities: The Peak of Web 1.0 (1994 - 2009)

GeoCities was the most famous Web 1.0 platform. It let users build personal websites without knowing coding - you filled in templates and they generated HTML. At its peak, GeoCities had 38 million user pages and was the third most visited website on the internet.

🌐 Netscape Navigator - http://www.geocities.com/Hollywood/Lot/5432/
⭐⭐ WELCOME TO PRIYA'S HOME PAGE ⭐⭐
🚧 THIS SITE IS UNDER CONSTRUCTION! 🚧
Hi! My name is Priya Sharma from Mumbai! I love cricket and Bollywood!
My favourite movies: DDLJ, Kuch Kuch Hota Hai, Lagaan

Hobby: CricketFav. Food: Pani Puri

This is Web 1.0 in its purest form. Notice: Priya wrote everything herself. Nobody can comment. Nobody can like. Nobody can see what others said. Visitors leave no trace. Priya updates it by re-uploading HTML via FTP. And in 2009, Yahoo (who bought GeoCities) shut it down - and every single one of those 38 million pages disappeared forever. Priya didn't own her website. GeoCities did.

⚠️ This is Why Blockchain Matters

GeoCities deleted 38 million personal websites overnight. The users created all the content. The platform owned all of it. On a decentralized web (like what blockchain enables), content is stored on distributed nodes worldwide - no single company can delete it. This is the promise of platforms like Arweave and IPFS that MST Blockchain integrates with.

Beginner-Friendly Deep Explanation

4.1 - What "Read-Only" Actually Means

The term "read-only" comes from computing. A read-only file is one you can open and view, but cannot modify. Web 1.0 treated the entire internet like a read-only library - an enormous collection of documents you could read, but could not write back to.

📚 Web 1.0 (Read-Only)
  • You can READ articles
  • You CANNOT write comments
  • You CANNOT upload content
  • You CANNOT create a profile
  • You CANNOT rate or like anything
  • Every visitor sees identical content
  • Content changes only when owner updates it
  • No database - just static files
✍️ Web 2.0 (Read-Write)
  • You can READ content
  • You CAN post comments
  • You CAN upload photos/videos
  • You CAN create and own a profile
  • You CAN like, rate, share content
  • Every visitor sees personalized content
  • Content updates in real-time by users
  • Database stores all user actions

4.2 - What is a Static Website?

A static website is one where every visitor gets the exact same content - like a printed pamphlet. The content only changes when the website owner manually edits the HTML file and re-uploads it. There is no database, no user accounts, no personalization.

🍕 Zomato Analogy

Imagine Zomato as a Web 1.0 website. You visit zomato.com/mumbai/restaurants and you see a list of restaurants - the SAME list every visitor sees, updated only when Zomato's developer manually edits the HTML file. There are no reviews, no ratings, no ordering. Just a list.

Your friend visits the same URL. Sees the exact same list. No personalization. No "Based on your order history." No "Most ordered near you." Just a static list that hasn't changed since last week.

This is Web 1.0. This is why Zomato as it actually exists - dynamic, personalized, interactive - could NOT have been built on Web 1.0 technology.

4.3 - What HTML Actually Looks Like

HTML is not a programming language - it is a markup language. It describes the STRUCTURE of a document, not logic or behaviour. It has no conditions (if/else), no loops, no functions. It just says: "This is a heading. This is a paragraph. This is an image."

💻 A Real Web 1.0 Page - 1998 Style
  <!-- This is a Web 1.0 HTML page. It looks exactly like this to ALL visitors. -->

  <HTML>
  <HEAD>
      <TITLE>Raj's Cricket Page</TITLE>
  </HEAD>
  <BODY BGCOLOR="yellow">

      <H1>Welcome to Raj's Cricket Page!</H1>
      <HR>  <!-- horizontal line -->
      
      <P>My name is Raj. I love cricket!</P>
      <P>India is the best team in the world.</P>
      
      <H2>My Favourite Players</H2>
      <UL>
          <LI>Sachin Tendulkar</LI>
          <LI>Sourav Ganguly</LI>
          <LI>Anil Kumble</LI>
      </UL>
      
      <IMG SRC="sachin.jpg" WIDTH=200>
      
      <HR>
      <P>Last Updated: February 14, 1998</P>
      
      <!-- NO database. NO user accounts. NO comments. NO ratings. -->
      <!-- Just text and images. Static forever. -->

  </BODY>
  </HTML>

  Result: Sachin retires in 2013. Raj's page still says "My Favourite Players."
  Nobody can update it except Raj. And Raj forgot his FTP password.

4.4 - CGI: Web 1.0's Attempt at Interactivity

By the mid-1990s, developers wanted some interactivity - at minimum, a form where users could submit their email. Enter CGI (Common Gateway Interface), which let servers run small scripts when a form was submitted. But even CGI was fundamentally limited: the script ran, generated a HTML response, and that was it. Nothing was stored. Nothing was personalized. The next visitor got the same static page.

💻 How CGI Worked (Simplified)

User fills a "Contact Us" form → Server runs a Perl script → Script sends an email to the website owner → Server shows a "Thank you!" page. No database. No record of what the user submitted. If the website owner checked their email a week later, they might see the message. Or not. This is why "Contact Us" forms on old websites were notoriously unreliable.

🔄 Every Web 1.0 Request - The Same Every Time
  FIRST VISIT (Priya, Monday):
  Browser → GET /sachin.html → Server reads file → Sends HTML → Browser shows page
  
  ─────────────────────────────────────────────────────────
  
  SECOND VISIT (Raj, Tuesday):
  Browser → GET /sachin.html → Server reads SAME file → Sends SAME HTML → Browser shows SAME page
  
  ─────────────────────────────────────────────────────────
  
  THIRD VISIT (Rahul, 3 months later):
  Browser → GET /sachin.html → Server reads SAME file → Sends SAME HTML → SAME page
  (Even if Sachin scored a century yesterday — page still says the same things)
  
  ─────────────────────────────────────────────────────────
  
  THE SERVER'S ONLY JOB:
  
  while (request_comes_in):
      read_file_from_disk(requested_url)
      send_file_to_browser()
      # That's literally it. No logic. No personalisation. No memory.

  Analogy: The server is a librarian who has ONE copy of each book.
  Everyone who asks for book X gets the same book X.
  The librarian does not know who asked, does not remember previous visitors,
  does not change the book based on who's reading it.
  That is Web 1.0.

The Web Evolution Timeline (Web 1.0 → 2.0 → 3.0)

This timeline is the backbone of everything in Module 1. Master it completely - it will appear in assessments, interviews, and grant proposals at MasterStroke Academy.

📄
1989 - 1995 - The Birth
Tim Berners-Lee Invents the WWW
HTML, HTTP, and URLs created at CERN. Web goes public in 1991. First websites: plain text, academic research. No images, no formatting, no interactivity. Anyone with a server could publish.
🖥️
1993 - 1994 - The Browser Wars Begin
Mosaic Browser Makes the Web Visual
Mosaic (1993) is the first graphical web browser. For the first time, you can see images on web pages. The web becomes accessible to non-technical people. India's first internet connection: VSNL (1995). Yahoo launches as a human-curated directory (1994).
🏠
1994 - 1999 - Web 1.0 Peak
GeoCities, Amazon, eBay, and the Dot-Com Boom
GeoCities (1994): users build personal pages. Amazon (1995): static product catalog with a shopping cart - early dynamic elements. eBay (1995): auction listings, first real-time web application. Dot-com bubble inflates massively. Everyone believes every company needs a website.
💥
2000 - 2002 - The Crash
Dot-Com Bust - Most Web 1.0 Companies Die
$5 trillion in market value evaporates. 50% of dot-com companies gone by 2002. Survivors: Amazon, Google, eBay - the ones with real utility. Lesson: having a website is not a business model. India's internet growth pauses briefly before resuming.
💬
2003 - 2006 - The Transition
Web 2.0 Emerges - Users Start Creating
MySpace (2003), Facebook (2004), YouTube (2005), Twitter (2006). O'Reilly coins "Web 2.0" at a conference. Dynamic pages, user accounts, databases, AJAX (pages update without refreshing). Web becomes two-way. India's broadband begins growing.
📱
2007 - 2016 - Web 2.0 Dominance
Smartphone Era - Platforms Become Monopolies
iPhone (2007) moves web to mobile. Facebook, Google, Amazon reach monopoly scale. Swiggy (2014), Uber India (2013), Ola (2010) launch. Jio launches (2016): 400M Indians get affordable internet. WhatsApp becomes India's primary communication platform. Platform lock-in complete.
🔗
2008 - Present - The Alternative
Web 3.0 / Blockchain - Decentralized Ownership
Bitcoin (2008), Ethereum (2015), MST Blockchain. The key difference from Web 2.0: users don't just create content - they OWN it. Smart contracts enforce rules without middlemen. No platform can delete your content or change the terms. India becomes world's largest crypto user base by 2022.

Real-World Case Studies

Case Study 1 - Rediff.com: India's Web 1.0 Giant

Rediff.com launched in 1996 and was the dominant Indian website through the early 2000s. At its peak, it had India's most visited news, email, and shopping portal. It was quintessential Web 1.0: a small team of editors and developers created all the content, users could only read, and monetization was entirely through banner advertisements.

Real-Time Updates❌ Page refreshed only when editor publishes✅ Live scores, live news, push notifications
Social Sharing❌ Copy URL manually✅ WhatsApp, Twitter, Facebook share buttons

Case Study 2 - Wikipedia: Web 1.0 Philosophy, Web 2.0 Technology

Wikipedia is a fascinating case because it uses Web 2.0 technology (anyone can edit) but follows Web 1.0 philosophy (no ads, no data collection, no algorithm). It launched in 2001 - right at the transition from Web 1.0 to Web 2.0 - and it represents what could have been: an interactive internet without surveillance capitalism.

💡 The Wikipedia Model and Blockchain

Wikipedia shows decentralized content curation can work: volunteers worldwide maintain millions of articles, no single company owns it (Wikipedia Foundation is a non-profit), and it runs on donations, not advertising. Blockchain takes this further - instead of trusting the Wikipedia Foundation to be honest, smart contracts encode the rules of governance mathematically, and contributors can be incentivized with tokens. Imagine Wikipedia where editors earn MST tokens for quality contributions.

Case Study 3 - Craigslist: The Web 1.0 Survivor

Craigslist launched in 1995 and still looks almost identical today - plain HTML, minimal design, no algorithm, no recommendations, no social features. Yet it handles 80 million classified ads per month and earns $700 million+ annually. Why did it survive when GeoCities, Excite, AltaVista, and hundreds of other Web 1.0 sites died?

Because classifieds don't need social features. A buyer wants to find a used bike. A seller wants to sell a used bike. A simple list works perfectly. No algorithm needed. No comments needed. No profile needed. Not every problem needs the full power of Web 2.0 - and this is true for blockchain too: not every problem needs a blockchain.

Assessment Preparation - Must-Know Facts

  • Web 1.0 = Read-Only. Users could only consume content, not create or interact. Like a printed newspaper on a screen.
  • Static websites serve identical content to every visitor. No personalization, no user accounts, no database.
  • Web 1.0 era: roughly 1991 to 2003. Started with WWW invention; ended as Web 2.0 platforms emerged.
  • Three core Web 1.0 technologies: HTML, HTTP, URLs. All invented by Tim Berners-Lee at CERN in 1989 - 1991.
  • HTML is a markup language, NOT a programming language. It describes structure (headings, paragraphs, images) - no logic, no conditions, no loops.
  • FTP (File Transfer Protocol) was how developers uploaded HTML files to web servers. Manual, slow, tedious.
  • CGI (Common Gateway Interface) was Web 1.0's attempt at interactivity - running scripts on server for form submissions. Nothing was stored persistently.
  • GeoCities (1994 - 2009) was the largest Web 1.0 platform - 38 million personal pages. Shut down in 2009, deleting everything. Users owned nothing.
  • Web 1.0 was paradoxically more decentralized than Web 2.0 - anyone could run a server and publish. Web 2.0 re-centralized on platforms like Facebook and YouTube.
  • Blockchain is Web 3.0 - combining Web 2.0's interactivity with Web 1.0's openness, but adding user ownership and decentralized control that neither era had.
Section 09

💼
Industry Insights

9.1 - "The Wayback Machine" - Preserving Web 1.0 History

The Internet Archive (archive.org) runs the "Wayback Machine" - a service that has crawled and saved copies of billions of websites since 1996. You can visit web.archive.org and see exactly what Rediff.com, GeoCities, or even the MST Blockchain website looked like in any past year. This is centralized historical preservation. Blockchain offers a better model: content permanently stored on-chain cannot be modified or deleted - ever - without requiring a trust in any organization like the Internet Archive.

9.2 - India's Web 1.0 Entrepreneurs Got Nothing

Thousands of Indian entrepreneurs built websites in 1999 - 2001 during the dot-com boom. Most used GeoCities, Tripod, or Angelfire to host their content - platforms that owned everything. When the bubble burst and these platforms shut down or pivoted, all that content was lost. The entrepreneurs had built on rented land. Blockchain's ownership model means builders own their work - smart contracts, tokens, and on-chain content persist regardless of what any platform does.

9.3 - Why Web 1.0 Economics Failed

Web 1.0 had no business model except banner advertising - and banner ads were terrible. Click-through rates were 44% in 1994 and fell to 0.1% by 2003. Content creators got nothing. Platform operators struggled. The entire ecosystem was waiting for a better model. Web 2.0 provided one (data-driven targeted advertising) - but at the cost of user privacy. Blockchain's token economics offer a third path: users pay creators directly, in micro-amounts, without advertising.

Web 1.0: Anyone could publish, but nobody could earn.
Web 2.0: Anyone could earn, but platforms owned everything.
Web 3.0: Anyone can publish, earn, AND own.

Glossary / Key Terms

Web 1.0

First era of the World Wide Web (1991 - 2003). Read-only, static content, no user interaction, no databases, no accounts.

Static Website

A website serving identical HTML files to every visitor. Content changes only when the developer manually updates and re-uploads files.

HTML

HyperText Markup Language. A markup language (NOT programming language) that describes document structure: headings, paragraphs, images, links.

HTTP

HyperText Transfer Protocol. Rules for requesting and delivering HTML documents over the internet. "https://" in your browser is HTTP + encryption.

URL

Uniform Resource Locator. A unique address for every document on the web. e.g., https://www.mstblockchain.com/whitepaper

FTP

File Transfer Protocol. How developers uploaded HTML files from their computers to web servers. Now largely replaced by Git, SSH, and CI/CD pipelines.

CGI

Common Gateway Interface. A way to run scripts on a web server when a form is submitted. Web 1.0's rudimentary attempt at interactivity.

GeoCities

Pioneering Web 1.0 platform (1994 - 2009). Free personal website hosting, 38M users at peak. Deleted everything when shut down - users owned nothing.

Dynamic Website

A website that generates personalized content per user based on database queries. Requires server-side programming and a database. Web 2.0 standard.

Dot-Com Boom/Bust

1997 - 2000 bubble where internet companies were overvalued; 2000 - 2002 crash where $5 trillion vanished and 50% of companies folded.

World Wide Web (WWW)

System of interlinked HTML documents accessed via browsers using HTTP. Invented 1989, public 1991. NOT the same as the internet.

Markup Language

A language that annotates text with tags to describe structure (e.g., this is a heading, this is bold). HTML is markup. Python, JavaScript are programming languages.

Question 1 MCQ 1 Mark Easy
Which of the following best describes a "static website" in the context of Web 1.0?
  • A
    A website that loads very slowly due to large images
  • B
    A website where every visitor receives the exact same HTML content, which only changes when the developer manually updates the file
  • C
    A website that doesn't use JavaScript or CSS for styling
  • D
    A website that works without an internet connection
✅ Correct Answer: B

Explanation: A static website serves the exact same HTML file to every visitor, regardless of who they are, what time it is, or what they've done before. The content changes only when the owner edits the HTML file and re-uploads it (via FTP). There is no database, no personalization, no user accounts. Option A confuses loading speed with static/dynamic nature. Option C is wrong - static sites can have CSS and basic JavaScript. Option D describes offline/cached apps. The critical defining feature is: identical content for all visitors, manually updated by the developer.

Question 2 MCQ 1 Mark Medium
Why is HTML described as a "markup language" rather than a "programming language"?
  • A
    Because HTML was invented before programming languages existed
  • B
    Because HTML runs slower than programming languages like Python or Java
  • C
    Because HTML only describes the structure and appearance of content using tags, but has no logic - no conditions, no loops, no functions
  • D
    Because HTML can only be read by browsers, not by servers
✅ Correct Answer: C

Explanation: The defining characteristic of a programming language is that it can express logic - decisions (if/else), repetition (loops), and reusable functions. HTML does none of these. It simply annotates text with tags: <h1> means "this is a main heading," <p> means "this is a paragraph," <img> means "display an image." HTML cannot say "if the user is from Mumbai, show biryani recipes." It can only describe what content looks like, not how to compute it. This distinction is crucial for blockchain developers - Solidity IS a programming language with full logic, conditions, and functions. HTML is just description.

Question 3 MCQ 1 Mark Medium
GeoCities was shut down by Yahoo in 2009, deleting 38 million personal websites. Which of the following blockchain concepts directly addresses this problem?
  • A
    Proof of Stake consensus - validators ensure websites are always available
  • B
    Gas fees - users pay to store their website permanently
  • C
    Decentralized storage (IPFS, Arweave) - content stored across distributed nodes worldwide, no single company can delete it
  • D
    Smart contracts - automated agreements that prevent companies from shutting down
✅ Correct Answer: C

Explanation: GeoCities' fatal flaw was that user content was stored on Yahoo's centralized servers. Yahoo decided to delete them - users had no recourse. Decentralized storage protocols like IPFS (InterPlanetary File System) and Arweave store content across thousands of nodes worldwide. To delete content from IPFS, you'd need to simultaneously remove it from every single node - computationally and practically impossible if the network is large enough. MST Blockchain integrates with such decentralized storage, enabling permanent, censorship-resistant content hosting. Option A (consensus) relates to transaction validation, not storage. Option D (smart contracts) can't force a company to keep servers running.

Question 4 MCQ 1 Mark Hard
In what way was Web 1.0 actually MORE decentralized than Web 2.0?
  • A
    Web 1.0 used peer-to-peer protocols, while Web 2.0 uses client-server models
  • B
    In Web 1.0, anyone could run their own server and publish freely; Web 2.0 funneled everyone onto a few dominant platforms (Facebook, YouTube, Instagram) that control publishing
  • C
    Web 1.0 had no central servers at all; Web 2.0 introduced centralized data centers
  • D
    Web 1.0 was decentralized in ownership because users owned their browsers
✅ Correct Answer: B

Explanation: This is a nuanced but critical question. In Web 1.0, publishing was distributed - anyone with a web server could publish anything to anyone in the world without asking permission from any platform. Thousands of independent websites competed. In Web 2.0, paradoxically, content creation became MORE centralized: instead of owning your own server, you publish through Facebook, YouTube, or Instagram - who then control your audience, your monetization, and your content's visibility. If Instagram bans you, your audience is gone. This is why the blockchain/Web3 movement is described as "a return to Web 1.0's openness, with Web 2.0's interactivity, plus real ownership." Option A is incorrect - both eras use client-server. Option C is false - Web 1.0 had servers, just many distributed ones.

Model Justification (Full 2 Marks):

While Web 2.0 brought genuine improvements - interactivity, ease of publishing, richer experiences, social connection - it also introduced serious new problems that Web 1.0 did not have. First, Web 1.0 collected essentially no user data, while Web 2.0 platforms track every click, build behavioral profiles, and sell them to advertisers - a significant privacy regression. Second, Web 1.0 was genuinely decentralized in publishing: anyone with a server could publish freely. Web 2.0 funneled billions of people into platforms controlled by a handful of corporations (Meta, Google, Amazon), giving these companies unprecedented censorship power. Third, Web 1.0 creators owned their content outright - when GeoCities shut down, it was tragic, but it was also possible to run your own server and not be dependent on any platform. Web 2.0 creators are entirely dependent on platform algorithms, policies, and goodwill for their income and reach. The transition was a trade of openness and privacy for convenience and interactivity - not a purely positive development.
