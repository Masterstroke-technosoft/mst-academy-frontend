04   Beginner-Friendly Explanation

The Three Contracts You'll Deploy - And How They Talk

🏛️ The Analogy

Think of your DAO as a town hall. The Governance Token is the voter ID - it proves you're a citizen and how much voting power you have. The Governor contract is the town hall itself - it accepts proposals, counts votes, and announces results. The TimelockController is the safe that holds the town's money and can only be opened after a mandatory waiting period, giving everyone time to object. When a proposal passes, the Timelock waits, then automatically executes the town's decision.

Your project will deploy and connect these three contracts, then run a complete governance cycle. Here's what you'll build:

T
TOKEN
GovernanceToken.sol (ERC20Votes)
An ERC-20 token with voting extensions. Holders can delegate their voting power to themselves or others. The token supply is minted to the deployer initially. It implements ERC20Votes, which tracks historical balances for snapshot-based voting.
G
GOVERNOR
MyDAO.sol (Governor + TimelockControl)
The Governor contract, inheriting from OpenZeppelin's Governor, GovernorCountingSimple, GovernorVotes, GovernorVotesQuorumFraction, and GovernorTimelockControl. It manages the proposal lifecycle: creating proposals, voting, queuing, and execution.
L
TIMELOCK
TimelockController (deployed separately)
The OpenZeppelin TimelockController contract. It enforces a minimum delay between a proposal passing and its execution. The Governor is granted the proposer and executor roles on the timelock.

🪙 Step 4: Write the Governance Token (GovernanceToken.sol) 
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Votes.sol";

contract GovernanceToken is ERC20Votes {
    constructor() ERC20("MyDAO Token", "MDT") {
        _mint(msg.sender, 1_000_000 * 10**decimals());
    }

    // Overrides required by Solidity
    function _update(address from, address to, uint256 value) internal override(ERC20Votes) {
        super._update(from, to, value);
    }

    function nonces(address owner) public view override returns (uint256) {
        return super.nonces(owner);
    }
}

This token mints 1,000,000 MDT to the deployer and supports delegation and snapshot voting via ERC20Votes.

🏛️ Step 5: Write the Governor Contract (MyDAO.sol) 
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/governance/Governor.sol";
import "@openzeppelin/contracts/governance/extensions/GovernorCountingSimple.sol";
import "@openzeppelin/contracts/governance/extensions/GovernorVotes.sol";
import "@openzeppelin/contracts/governance/extensions/GovernorVotesQuorumFraction.sol";
import "@openzeppelin/contracts/governance/extensions/GovernorTimelockControl.sol";

contract MyDAO is Governor, GovernorCountingSimple, GovernorVotes, GovernorVotesQuorumFraction, GovernorTimelockControl {
    constructor(IVotes _token, TimelockController _timelock)
        Governor("MyDAO")
        GovernorVotes(_token)
        GovernorVotesQuorumFraction(4) // 4% quorum
        GovernorTimelockControl(_timelock)
    {}

    function votingDelay() public pure override returns (uint256) { return 1 days; }
    function votingPeriod() public pure override returns (uint256) { return 3 days; }
    function proposalThreshold() public pure override returns (uint256) { return 10_000e18; }

    function quorum(uint256 blockNumber) public view override(Governor, GovernorVotesQuorumFraction) returns (uint256) {
        return super.quorum(blockNumber);
    }
    function state(uint256 proposalId) public view override(Governor, GovernorTimelockControl) returns (ProposalState) {
        return super.state(proposalId);
    }
    function _execute(uint256 proposalId, address[] memory targets, uint256[] memory values, bytes[] memory calldatas, bytes32 descriptionHash) internal override(Governor, GovernorTimelockControl) {
        super._execute(proposalId, targets, values, calldatas, descriptionHash);
    }
    function _cancel(address[] memory targets, uint256[] memory values, bytes[] memory calldatas, bytes32 descriptionHash) internal override(Governor, GovernorTimelockControl) returns (uint256) {
        return super._cancel(targets, values, calldatas, descriptionHash);
    }
    function _executor() internal view override(Governor, GovernorTimelockControl) returns (address) {
        return super._executor();
    }
    function supportsInterface(bytes4 interfaceId) public view override(Governor, GovernorTimelockControl) returns (bool) {
        return super.supportsInterface(interfaceId);
    }
}

The parameters: 1-day voting delay, 3-day voting period, 4% quorum, and 10,000 token proposal threshold.

📜 Step 6: Write the Deployment Script (scripts/deploy.js) 
const { ethers } = require("hardhat");

async function main() {
  // 1. Deploy GovernanceToken
  const Token = await ethers.getContractFactory("GovernanceToken");
  const token = await Token.deploy();
  await token.waitForDeployment();
  console.log("Token deployed to:", token.target);

  // 2. Deploy TimelockController (minDelay = 2 days, proposers & executors initially empty)
  const Timelock = await ethers.getContractFactory("TimelockController");
  const timelock = await Timelock.deploy(2 * 24 * 3600, [], [], ethers.ZeroAddress);
  await timelock.waitForDeployment();
  console.log("Timelock deployed to:", timelock.target);

  // 3. Deploy Governor
  const MyDAO = await ethers.getContractFactory("MyDAO");
  const governor = await MyDAO.deploy(token.target, timelock.target);
  await governor.waitForDeployment();
  console.log("Governor deployed to:", governor.target);

  // 4. Grant roles on timelock
  const PROPOSER_ROLE = ethers.keccak256(ethers.toUtf8Bytes("PROPOSER_ROLE"));
  const EXECUTOR_ROLE = ethers.keccak256(ethers.toUtf8Bytes("EXECUTOR_ROLE"));
  const CANCELLER_ROLE = ethers.keccak256(ethers.toUtf8Bytes("CANCELLER_ROLE"));
  await timelock.grantRole(PROPOSER_ROLE, governor.target);
  await timelock.grantRole(EXECUTOR_ROLE, governor.target);
  await timelock.grantRole(CANCELLER_ROLE, governor.target);
  console.log("Roles granted to governor");

  // 5. Delegate voting power (optional, but needed for proposal threshold)
  // The deployer automatically has tokens, but must delegate to themselves to have voting power.
  const signer = await ethers.provider.getSigner();
  await token.delegate(signer.address);
  console.log("Delegated tokens to deployer");

  console.log("\nDeployment complete! DAO is live on MST Testnet.");
  console.log("Token:", token.target);
  console.log("Timelock:", timelock.target);
  console.log("Governor:", governor.target);
}

main().catch(console.error);

06   Proposal Lifecycle Walkthrough

From Idea to Execution - Run Your DAO

Now that your DAO is deployed, let's run a proposal through the entire lifecycle. You'll need the Governor address, the token address, and a target contract to interact with (we'll use a simple Box contract that stores a value).

📝 Step 8: Deploy a Target Contract (Box.sol) 
// contracts/Box.sol
contract Box {
    uint256 public value;
    function store(uint256 _value) external {
        value = _value;
    }
}

Deploy it to MST Testnet and note the address.

🗳️ Step 9: Create a Proposal to Change Box's Value

Write a script scripts/propose.js:

const governor = await ethers.getContractAt("MyDAO", governorAddress);
const targets = [boxAddress];
const values = [0];
const calldatas = [new ethers.Interface(["function store(uint256)"]).encodeFunctionData("store", [42])];
const description = "Proposal #1: Change Box value to 42";
const tx = await governor.propose(targets, values, calldatas, description);
const receipt = await tx.wait();
const proposalId = receipt.logs[0].topics[1]; // proposal ID
console.log("Proposal created with ID:", proposalId);

Run the script. The proposal enters the Pending state. Voting delay is 1 day (86400 seconds). You can fast-forward time using Hardhat's evm_increaseTime for testing.

✅ Step 10: Vote on the Proposal

After the voting delay, the proposal becomes Active. Token holders can vote:

const tx = await governor.castVote(proposalId, 1); // 1 = For
await tx.wait();

Wait for the voting period to end (or advance time). Then check the state:

const state = await governor.state(proposalId);
console.log("State:", state); // 4 = Succeeded
⏳ Step 11: Queue in Timelock 
const descriptionHash = ethers.keccak256(ethers.toUtf8Bytes(description));
const tx = await governor.queue(targets, values, calldatas, descriptionHash);
await tx.wait();
🚀 Step 12: Execute

After the timelock delay (2 days), anyone can execute:

const tx = await governor.execute(targets, values, calldatas, descriptionHash);
await tx.wait();
// Check Box value:
const box = await ethers.getContractAt("Box", boxAddress);
const val = await box.value();
console.log("Box value:", val.toString()); // 42

07   Testing Your DAO

Prove Your DAO Works Before Anyone Uses It

Write comprehensive Hardhat tests that simulate the entire lifecycle using time manipulation. Here's a skeleton:

const { expect } = require("chai");
const { time } = require("@nomicfoundation/hardhat-network-helpers");

describe("MyDAO", function () {
  let token, timelock, governor, box;
  let deployer, voter1, voter2;

  beforeEach(async function () {
    [deployer, voter1, voter2] = await ethers.getSigners();
    // Deploy token
    const Token = await ethers.getContractFactory("GovernanceToken");
    token = await Token.deploy();
    // Deploy timelock
    const Timelock = await ethers.getContractFactory("TimelockController");
    timelock = await Timelock.deploy(2 * 24 * 3600, [], [], ethers.ZeroAddress);
    // Deploy governor
    const Governor = await ethers.getContractFactory("MyDAO");
    governor = await Governor.deploy(token.target, timelock.target);
    // Grant roles
    const PROPOSER_ROLE = ethers.keccak256(ethers.toUtf8Bytes("PROPOSER_ROLE"));
    const EXECUTOR_ROLE = ethers.keccak256(ethers.toUtf8Bytes("EXECUTOR_ROLE"));
    await timelock.grantRole(PROPOSER_ROLE, governor.target);
    await timelock.grantRole(EXECUTOR_ROLE, governor.target);
    // Deploy Box
    const Box = await ethers.getContractFactory("Box");
    box = await Box.deploy();
    // Delegate tokens to deployer
    await token.delegate(deployer.address);
    // Transfer some tokens to voter1 for testing
    await token.transfer(voter1.address, ethers.parseEther("50000"));
    await token.connect(voter1).delegate(voter1.address);
  });

  it("Should create a proposal, vote, queue, and execute", async function () {
    const targets = [box.target];
    const values = [0];
    const calldatas = [box.interface.encodeFunctionData("store", [42])];
    const description = "Set Box to 42";

    const tx = await governor.propose(targets, values, calldatas, description);
    const receipt = await tx.wait();
    const proposalId = receipt.logs[0].topics[1];

    // Advance time past voting delay
    await time.increase(86400 + 1); // 1 day + 1 sec
    await governor.castVote(proposalId, 1); // deployer votes For
    await governor.connect(voter1).castVote(proposalId, 1); // voter1 votes For

    // Advance past voting period
    await time.increase(3 * 86400 + 1);
    expect(await governor.state(proposalId)).to.equal(4); // Succeeded

    // Queue
    const descHash = ethers.keccak256(ethers.toUtf8Bytes(description));
    await governor.queue(targets, values, calldatas, descHash);
    expect(await governor.state(proposalId)).to.equal(5); // Queued

    // Advance past timelock
    await time.increase(2 * 86400 + 1);
    await governor.execute(targets, values, calldatas, descHash);
    expect(await governor.state(proposalId)).to.equal(7); // Executed

    expect(await box.value()).to.equal(42);
  });
});

Run the tests: npx hardhat test. They should pass in seconds, simulating 6 days of real time in milliseconds.

10   Common Pitfalls

Avoid These Mistakes When Building Your DAO

⚠️
Forgetting to delegate tokens before voting
In ERC20Votes, simply holding tokens does not give you voting power. You must call delegate(address) - either to yourself or to a representative. Many DAOs have failed proposals because holders forgot to delegate.
⚠️
Not granting the governor proposer/executor roles on the timelock
If the governor can't propose (queue) or execute on the timelock, proposals will be stuck in Succeeded state forever. Always grant the PROPOSER_ROLE and EXECUTOR_ROLE to the governor address after deployment.
⚠️
Setting the timelock minDelay too short
A very short timelock (e.g., 1 hour) defeats the purpose - token holders have no time to exit. Use at least 24 hours for a meaningful delay. For testing on MST, you can use shorter delays, but for production, longer is safer.

13   Glossary

Key Terms - Sub-Module 15.5

ERC20Votes
An extension of ERC-20 that enables delegation of voting power and snapshot-based governance.
Governor Contract
OpenZeppelin's modular contract that manages the proposal lifecycle: creation, voting, queuing, and execution.
TimelockController
A contract that enforces a mandatory delay between proposal approval and execution, providing a safety window.
Proposal Threshold
The minimum number of tokens a user must hold (or have delegated) to submit a proposal.
Quorum
The minimum number of votes (or % of total supply) that must be cast for a proposal to be valid.
Snapshot
The block number at which token balances are recorded to determine voting power for a proposal.
