03   Project Planning

What Are We Building? Requirements & Architecture

Before writing a single line of code, we define exactly what our dApp must do. A clear specification prevents scope creep, ensures we build the right thing, and gives us a checklist for testing.

1
Functional Requirements
Create Task: Any user can add a new task with a text description. The task is stored on the blockchain with a unique ID.
Toggle Task: Any user can mark a task as completed or uncompleted by its ID.
View All Tasks: Anyone can read the complete list of tasks - no gas cost, no wallet required.
View Task Count: Anyone can see how many tasks exist.
Events: The contract emits events when tasks are created or toggled, enabling frontend reactivity.
2
Architecture Decisions
Smart Contract: A single Solidity contract (TodoList.sol) that manages all task data and logic. No ownership restrictions - anyone can add and toggle tasks for simplicity.
Data Structure: A struct for Task (id, content, completed) and a dynamic array to store all tasks.
Frontend: React with Vite for fast builds, ethers.js for blockchain interaction, and Tailwind CSS for styling.
Network: Deploy to MST Testnet (Chain ID 4545) for testing and demonstration.
3
Tech Stack
Smart Contract: Solidity 0.8.20, Hardhat framework
Testing: Hardhat + Chai + ethers.js
Frontend: React 18 + Vite + ethers.js + MST SDK
Deployment: Hardhat scripts to MST Testnet (RPC: https://testnetrpc.mstblockchain.com)
Version Control: Git + GitHub (public repository)

04   Smart Contract

The TodoList Smart Contract - Complete Code & Explanation

Below is the complete, production-ready TodoList smart contract. Every design decision is explained. Copy this code into contracts/TodoList.sol in your Hardhat project.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

/**
 * @title TodoList
 * @notice A decentralized to‑do list where anyone can add tasks and toggle completion.
 *         All data is stored permanently on the blockchain.
 * @dev Uses a dynamic array of Task structs. No access control for simplicity.
 */
contract TodoList {

    // ─── DATA STRUCTURE ───

    /**
     * @dev Represents a single task in the to‑do list.
     * @param id The unique identifier (index in the tasks array).
     * @param content The text description of the task.
     * @param completed Whether the task has been marked as completed.
     */
    struct Task {
        uint256 id;
        string content;
        bool completed;
    }

    // ─── STATE VARIABLES ───

    /// @notice The dynamic array storing all tasks. Public visibility auto‑generates a getter.
    Task[] public tasks;

    // ─── EVENTS ───

    /**
     * @notice Emitted when a new task is created.
     * @param id The unique identifier of the new task.
     * @param content The text description of the new task.
     */
    event TaskCreated(uint256 indexed id, string content);

    /**
     * @notice Emitted when a task's completion status is toggled.
     * @param id The unique identifier of the toggled task.
     * @param completed The new completion status.
     */
    event TaskToggled(uint256 indexed id, bool completed);

    // ─── FUNCTIONS ───

    /**
     * @notice Creates a new task and adds it to the list.
     * @param content The text description of the task (cannot be empty).
     * @return The ID of the newly created task.
     */
    function createTask(string memory content) public returns (uint256) {
        // Validate input
        require(bytes(content).length > 0, "Task content cannot be empty");
        require(bytes(content).length <= 280, "Task content too long (max 280 chars)");

        uint256 taskId = tasks.length;

        // Push a new Task struct to the array
        tasks.push(Task({
            id: taskId,
            content: content,
            completed: false
        }));

        // Emit event for frontend to listen
        emit TaskCreated(taskId, content);

        return taskId;
    }

    /**
     * @notice Toggles the completion status of a task by its ID.
     * @param taskId The ID of the task to toggle.
     */
    function toggleCompleted(uint256 taskId) public {
        // Validate that the task exists
        require(taskId < tasks.length, "Task does not exist");

        // Toggle the completed flag
        tasks[taskId].completed = !tasks[taskId].completed;

        // Emit event for frontend to listen
        emit TaskToggled(taskId, tasks[taskId].completed);
    }

    /**
     * @notice Returns the total number of tasks.
     * @return The current length of the tasks array.
     */
    function getTaskCount() public view returns (uint256) {
        return tasks.length;
    }

    /**
     * @notice Returns all tasks. Useful for frontend display.
     * @return An array of all Task structs.
     * @dev Be careful: if the array is very large, this may hit gas limits when called on‑chain.
     *      For this project, the array is small. For production, consider pagination.
     */
    function getAllTasks() public view returns (Task[] memory) {
        return tasks;
    }
}
🔑 Design Decisions Explained

Why an array instead of a mapping? An array allows easy iteration over all tasks for display. A mapping would be more gas-efficient for lookups but would require additional tracking of keys for iteration. For a to-do list with a reasonable number of tasks, the array is the simpler, more readable choice.

Why no access control? This is an intentionally open contract - anyone can add tasks and toggle any task. This keeps the project focused on the core concepts without the complexity of ownership. In a production app, you would likely restrict task modification to the task creator using msg.sender tracking.

Why events? Events allow the frontend to react to blockchain state changes without polling. When a user adds a task, the TaskCreated event fires, and the frontend listener updates the UI instantly - providing a Web2-like user experience.

05   Testing

Writing a Comprehensive Test Suite

Testing smart contracts is not optional - it is the most important part of development. Smart contracts handle real money and cannot be patched after deployment. Every function, every edge case, every revert condition must be tested. Here is the complete test file for our TodoList contract.

// test/TodoList.test.js
const { expect } = require("chai");
const { ethers } = require("hardhat");

describe("TodoList Contract", function () {
    let TodoList;
    let todoList;
    let owner;
    let addr1;

    // Deploy a fresh contract before each test
    beforeEach(async function () {
        TodoList = await ethers.getContractFactory("TodoList");
        [owner, addr1] = await ethers.getSigners();
        todoList = await TodoList.deploy();
        await todoList.waitForDeployment();
    });

    describe("Deployment", function () {
        it("Should start with zero tasks", async function () {
            expect(await todoList.getTaskCount()).to.equal(0);
        });

        it("Should return an empty array for getAllTasks", async function () {
            const allTasks = await todoList.getAllTasks();
            expect(allTasks.length).to.equal(0);
        });
    });

    describe("createTask", function () {
        it("Should create a task and increment the count", async function () {
            await todoList.createTask("Learn Solidity");
            expect(await todoList.getTaskCount()).to.equal(1);
        });

        it("Should store the correct task content", async function () {
            await todoList.createTask("Build a dApp");
            const task = await todoList.tasks(0);
            expect(task.content).to.equal("Build a dApp");
            expect(task.completed).to.equal(false);
            expect(task.id).to.equal(0);
        });

        it("Should emit a TaskCreated event", async function () {
            await expect(todoList.createTask("Write tests"))
                .to.emit(todoList, "TaskCreated")
                .withArgs(0, "Write tests");
        });

        it("Should revert if content is empty", async function () {
            await expect(
                todoList.createTask("")
            ).to.be.revertedWith("Task content cannot be empty");
        });

        it("Should revert if content exceeds 280 characters", async function () {
            const longString = "a".repeat(281);
            await expect(
                todoList.createTask(longString)
            ).to.be.revertedWith("Task content too long");
        });

        it("Should allow multiple tasks to be created", async function () {
            await todoList.createTask("Task 1");
            await todoList.createTask("Task 2");
            await todoList.createTask("Task 3");
            expect(await todoList.getTaskCount()).to.equal(3);
        });

        it("Should allow any address to create a task", async function () {
            await todoList.connect(addr1).createTask("Task from addr1");
            const task = await todoList.tasks(0);
            expect(task.content).to.equal("Task from addr1");
        });
    });

    describe("toggleCompleted", function () {
        beforeEach(async function () {
            await todoList.createTask("Test Task");
        });

        it("Should toggle completed from false to true", async function () {
            await todoList.toggleCompleted(0);
            const task = await todoList.tasks(0);
            expect(task.completed).to.equal(true);
        });

        it("Should toggle completed from true back to false", async function () {
            await todoList.toggleCompleted(0); // false → true
            await todoList.toggleCompleted(0); // true → false
            const task = await todoList.tasks(0);
            expect(task.completed).to.equal(false);
        });

        it("Should emit a TaskToggled event", async function () {
            await expect(todoList.toggleCompleted(0))
                .to.emit(todoList, "TaskToggled")
                .withArgs(0, true);
        });

        it("Should revert if task ID does not exist", async function () {
            await expect(
                todoList.toggleCompleted(999)
            ).to.be.revertedWith("Task does not exist");
        });

        it("Should allow any address to toggle a task", async function () {
            await todoList.connect(addr1).toggleCompleted(0);
            const task = await todoList.tasks(0);
            expect(task.completed).to.equal(true);
        });
    });

    describe("getAllTasks", function () {
        it("Should return all tasks in order", async function () {
            await todoList.createTask("First");
            await todoList.createTask("Second");
            await todoList.createTask("Third");

            const allTasks = await todoList.getAllTasks();
            expect(allTasks.length).to.equal(3);
            expect(allTasks[0].content).to.equal("First");
            expect(allTasks[1].content).to.equal("Second");
            expect(allTasks[2].content).to.equal("Third");
        });

        it("Should return tasks with correct completion status", async function () {
            await todoList.createTask("Task A");
            await todoList.createTask("Task B");
            await todoList.toggleCompleted(0);

            const allTasks = await todoList.getAllTasks();
            expect(allTasks[0].completed).to.equal(true);
            expect(allTasks[1].completed).to.equal(false);
        });
    });
});
💡 Testing Best Practices

Notice how each test is independent - the beforeEach block deploys a fresh contract for every test. This ensures tests don't interfere with each other. Also notice how we test both happy paths (normal usage) and sad paths (invalid inputs, reverts). Both are equally important.

06   Deployment

Deploying to the MST Testnet

With the contract written and tested, we deploy to the MST Testnet where real users can interact with it. Here is the deployment script and the Hardhat configuration.

// scripts/deploy.js
const hre = require("hardhat");

async function main() {
    console.log("Deploying TodoList to MST Testnet...");

    const TodoList = await hre.ethers.getContractFactory("TodoList");
    const todoList = await TodoList.deploy();
    await todoList.waitForDeployment();

    const address = await todoList.getAddress();
    console.log("TodoList deployed to:", address);
    console.log("Verify with: npx hardhat verify --network mst_testnet", address);
}

main().catch((error) => {
    console.error(error);
    process.exitCode = 1;
});

import { useState, useEffect } from 'react';
import { ethers } from 'ethers';
import TodoListABI from './abis/TodoList.json';

const CONTRACT_ADDRESS = "0x_YOUR_DEPLOYED_CONTRACT_ADDRESS_"; // Replace!

function App() {
    const [provider, setProvider] = useState(null);
    const [signer, setSigner] = useState(null);
    const [contract, setContract] = useState(null);
    const [account, setAccount] = useState('');
    const [tasks, setTasks] = useState([]);
    const [newTask, setNewTask] = useState('');
    const [loading, setLoading] = useState(false);

    // Connect wallet
    const connectWallet = async () => {
        if (window.ethereum) {
            const provider = new ethers.BrowserProvider(window.ethereum);
            const signer = await provider.getSigner();
            const address = await signer.getAddress();
            const contract = new ethers.Contract(CONTRACT_ADDRESS, TodoListABI.abi, signer);

            setProvider(provider);
            setSigner(signer);
            setContract(contract);
            setAccount(address);

            // Listen for account changes
            window.ethereum.on('accountsChanged', (accounts) => {
                if (accounts.length === 0) disconnectWallet();
                else setAccount(accounts[0]);
            });
        } else {
            alert('Please install MST BridgeKey or MetaMask!');
        }
    };

    const disconnectWallet = () => {
        setProvider(null); setSigner(null); setContract(null);
        setAccount(''); setTasks([]);
    };

    // Fetch all tasks
    const fetchTasks = async () => {
        if (!contract) return;
        const allTasks = await contract.getAllTasks();
        setTasks(allTasks.map(t => ({
            id: Number(t.id),
            content: t.content,
            completed: t.completed
        })));
    };

        e.preventDefault();

            setNewTask('');

            alert('Error: ' + (err.reason || err.message));
        }
        setLoading(false);
    };

    useEffect(() => {
        if (contract) fetchTasks();
    }, [contract]);

11   Glossary

Key Terms - Sub-Module 8.8

dApp
Decentralized Application - a frontend (web/mobile) connected to a smart contract backend on a blockchain.
ABI
Application Binary Interface - a JSON file that describes a contract's functions and parameters, used by ethers.js to interact with deployed contracts.
BrowserProvider
An ethers.js class that connects to the user's wallet (BridgeKey/MetaMask) via window.ethereum.
Signer
An ethers.js object representing a wallet that can sign and send transactions to the blockchain.
waitForDeployment
A Hardhat method that waits until a contract deployment transaction is confirmed on the blockchain.
Vite
A fast build tool and development server for modern JavaScript projects. Used as an alternative to Create React App.
MSTScan
The block explorer for MST Blockchain where you can view transactions, blocks, and verified contract source code.
beforeEach
A Mocha/Chai hook that runs before every test case, used to deploy a fresh contract for test isolation.
