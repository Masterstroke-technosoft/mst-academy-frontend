The Developer Who Coded Without npm - A Cautionary Tale

📖 Story

In 2021, a solo developer in Chennai began building a DeFi dashboard using React and ethers.js. He manually downloaded the ethers.js file, placed it in a `/lib` folder, and referenced it with a script tag. Everything worked until he needed to add the OpenZeppelin library for token contracts. In fifteen seconds, every library was installed, version conflicts were resolved, and the project compiled on the first try. The toolchain had transformed a nightmare into a command.

03   Background & History

Why JavaScript Conquered Blockchain Development

When Ethereum launched in 2015, smart contracts were written in Solidity, but the tooling around them was primitive. Developers compiled contracts in a browser-based IDE (Remix) and deployed them manually. There was no testing framework, no package manager, and no reproducible build pipeline. Every developer reinvented the wheel.

The breakthrough came in 2018 - 2019 with Hardhat (originally Buidler), a development environment built entirely on Node.js. Hardhat provided a local Ethereum network, a Solidity compiler, a testing framework, and a plugin system - all powered by Node.js and npm. Because Node.js uses JavaScript, and JavaScript was already the language of web frontends, the same developers who built dApp interfaces could now write their deployment scripts and tests in the same language. This convergence made blockchain development accessible to millions of JavaScript developers worldwide.

Today, the standard Web3 stack is Node.js + npm + Hardhat + VS Code. Whether you're building on Ethereum, Polygon, BSC, or MST - any EVM-compatible chain - you'll use the same tools. MST's full EVM compatibility means that every Hardhat plugin, every OpenZeppelin contract, and every npm library works out of the box. The toolchain you set up today will serve you across the entire blockchain ecosystem.

A Brief History of the Holy Trinity

2009
Node.js created by Ryan Dahl. First runtime to bring JavaScript to the server. Built on Chrome's V8 engine. Enabled non-blocking I/O, making it ideal for tools that interact with networks.
2010
npm launched by Isaac Schlueter. Originally "Node Package Manager". Now the world's largest software registry with over 2 million packages. Every Solidity tool lives on npm.
2015
Visual Studio Code released by Microsoft. Built on Electron (Node.js), open source, extensible. Quickly became the most popular editor for Web3 development due to its Solidity extension ecosystem.
2018
Hardhat (then Buidler) launched. Node.js-based Ethereum development environment. Defined the modern Solidity workflow: compile, test, deploy - all from JavaScript.
Today
MST fully compatible. MST's EVM compatibility means the exact same Node.js + Hardhat + VS Code toolchain works for MST Testnet and mainnet. Deploy a contract on MST with the same scripts you'd use on Ethereum - just change the RPC URL.
04   Beginner-Friendly Explanation

The Kitchen, the Recipe Book, and the Chef's Knife

🍳 The Analogy

Think of Node.js as the kitchen - the environment where code runs. npm is the recipe book and grocery delivery service - it fetches all the ingredients (libraries) you need, in the right amounts (versions), and ensures they don't conflict. Visual Studio Code is the chef's knife - the tool that makes writing, editing, and organizing code fast and precise. You can cook without them, but you'll spend more time fighting the environment than creating the dish.

Let's break down what each tool actually does:

N
Node.js
What is Node.js?
Node.js is an open-source, cross-platform JavaScript runtime environment. It allows you to run JavaScript code outside a browser - on your computer or a server. In blockchain development, Node.js runs Hardhat (which compiles Solidity), executes test scripts, and powers deployment scripts. It's the engine that processes all the JavaScript files you'll write to manage smart contracts. Node.js comes with a built-in package manager: npm.
P
npm
What is npm?
npm is the Node Package Manager. It is a command-line tool and an online registry of over 2 million packages. When you type npm install hardhat, npm downloads Hardhat and every library that Hardhat depends on, placing them in a node_modules folder. It also creates a package.json file that records your project's dependencies. Any developer can clone your project, run npm install, and instantly have the exact same environment. npm also lets you run scripts (like npm test or npm run compile) defined in package.json.
V
VS Code
What is Visual Studio Code?
Visual Studio Code is a free, open-source code editor developed by Microsoft. It runs on Windows, macOS, and Linux. Its power comes from extensions - thousands of community-built plugins that add language support, debugging, formatting, and linting. For Solidity development, the essential extensions are: Solidity by Juan Blanco (syntax highlighting, compilation, error checking), Prettier (automatic code formatting), and ESLint (code quality rules). VS Code also has a built-in terminal, so you can run npm commands without leaving the editor.

📥 Step 1: Install Node.js (includes npm)

Why Node.js? Hardhat, ethers.js, and most blockchain tools run on Node.js. It's the runtime that executes your deployment scripts and tests.

Installation: Go to nodejs.org and download the LTS (Long-Term Support) version. LTS is recommended for stability - currently Node 20.x or 22.x. The installer includes npm automatically.

Windows: Run the .msi installer. Accept the license, choose the default location, and ensure "Add to PATH" is checked. Restart your terminal after installation.

macOS: Use Homebrew: brew install node. Or download the .pkg from nodejs.org.

Linux (Ubuntu/Debian): Use the official NodeSource repository: curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs

Verify: Open a terminal (Command Prompt on Windows, Terminal on Mac/Linux) and run: node -v - should print a version like v20.11.0
npm -v - should print a version like 10.2.4

If both commands return version numbers, Node.js and npm are correctly installed.

🖥️ Step 2: Install Visual Studio Code

Why VS Code? It is the most popular editor among blockchain developers due to its rich extension ecosystem, built-in terminal, and Git integration.

Download: Visit code.visualstudio.com and download the installer for your operating system.

Installation: Run the installer. On Windows, check "Add to PATH" and "Open with Code" context menu options. On macOS, drag VS Code to the Applications folder.

Launch: Open VS Code. You'll see a Welcome page. We'll configure extensions in the next step.

🔌 Step 3: Install VS Code Extensions for Solidity

Extensions are the heart of VS Code's power. Click the Extensions icon on the left sidebar (or press Ctrl+Shift+X / Cmd+Shift+X) and search for each:

  • Solidity by Juan Blanco (solidity) - Syntax highlighting, compilation, code completion, error diagnostics. The most essential extension for .sol files. After installing, VS Code will automatically detect Solidity files and provide IntelliSense.
  • Prettier - Code formatter (esbenp.prettier-vscode) - Automatically formats your code on save. Configure it to format Solidity files by adding to settings: "[solidity]": { "editor.defaultFormatter": "esbenp.prettier-vscode" }
  • ESLint (dbaeumer.vscode-eslint) - Linting for JavaScript. Catches errors in your Hardhat scripts and tests before you run them.
  • GitLens (eamodio.gitlens) - Supercharges Git integration. Shows who last modified a line of code, commit history, and more - invaluable for collaborative projects.
  • Material Icon Theme (optional) - Adds beautiful icons for different file types (.sol, .js, .json), making the file explorer more readable.

After installing, reload VS Code (Ctrl+Shift+P → "Developer: Reload Window").

📁 Step 4: Create a Project & Initialize npm

Open VS Code's integrated terminal (Ctrl+` or View → Terminal).

mkdir my-dapp - create a new folder
cd my-dapp - enter the folder
npm init -y - initialize a new npm project with default values

This creates a package.json file, the manifest of your project. It records your project's name, version, scripts, and dependencies. You'll see output like:

{
  "name": "my-dapp",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}

We'll modify this file later to add scripts like "compile": "hardhat compile".

🔧 Step 5: Install Hardhat & ethers.js

In the terminal (still inside your project folder), run:

npm install --save-dev hardhat ethers

This single command does three things: 1. Downloads Hardhat and ethers.js from the npm registry. 2. Downloads all their dependencies (over 100 packages) recursively. 3. Saves them to a node_modules folder and records them as devDependencies in package.json.

Why --save-dev? Hardhat and ethers are development tools, not code that runs in production. Marking them as devDependencies keeps your production dependencies clean (useful when you later build a frontend).

After installation, your package.json will contain:

"devDependencies": {
  "ethers": "^6.13.0",
  "hardhat": "^2.22.0"
}

The caret (^) means "compatible with the specified version, allowing minor and patch updates". This is semantic versioning (SemVer) in action.

✅ Step 6: Write a Quick Connectivity Test

Create a new file in VS Code: test-connection.js. Paste the following code:

const { ethers } = require("ethers");

async function main() {
  // Connect to MST Testnet RPC
  const provider = new ethers.JsonRpcProvider("https://testnet-rpc.mstblockchain.com");
  // Get the latest block number
  const blockNumber = await provider.getBlockNumber();
  console.log("Connected to MST Testnet! Latest block:", blockNumber);
}

main().catch(console.error);

Run it from the terminal: node test-connection.js

You should see output like: Connected to MST Testnet! Latest block: 2847653

If you see a block number, congratulations! Your entire toolchain - Node.js, npm, ethers.js - is working and connected to the MST blockchain.

06   Deep Dive: npm & package.json

The Engine That Manages Your Dependencies

npm is far more than a simple installer. It is a complete dependency management system that solves the "dependency hell" problem that plagued software development for decades. Understanding its internals will save you countless hours of frustration.

The node_modules Folder and the Dependency Tree

When you run npm install hardhat, npm looks up "hardhat" in the npm registry, downloads the package, and places it in node_modules/hardhat. But Hardhat itself depends on other packages - solc (the Solidity compiler), mocha (test framework), chai (assertion library), and dozens more. npm recursively downloads all these transitive dependencies. The result is a node_modules folder that can contain hundreds of packages, even if you only directly installed one.

This recursive installation means you never have to manually track what a package depends on. npm handles it automatically. When a package is updated on the registry, you run npm update and npm resolves the new dependency tree, respecting version constraints.

Semantic Versioning (SemVer)

npm packages use a three-part version number: MAJOR.MINOR.PATCH (e.g., 2.22.0). The rules are:

  • PATCH (0.0.x): Backward-compatible bug fixes. e.g., 2.22.1 fixes a bug in 2.22.0 without changing the API.
  • MINOR (0.x.0): Backward-compatible new features. e.g., 2.23.0 adds a new function but doesn't break existing code.
  • MAJOR (x.0.0): Incompatible API changes. e.g., 3.0.0 might rename functions or remove deprecated features. Upgrading may require code changes.

In package.json, the caret (^) prefix means "allow updates that do not change the leftmost non-zero digit". So ^2.22.0 allows versions 2.x.x but not 3.0.0. The tilde (~) prefix is stricter: ~2.22.0 allows 2.22.x only. Pinning an exact version (2.22.0 without a prefix) ensures absolute reproducibility - critical for smart contract deployments where a version change could introduce a vulnerability.

package-lock.json - The Deterministic Install

When you run npm install, npm generates a package‑lock.json file. This file records the exact version of every package (and every transitive dependency) that was installed. When another developer clones your project and runs npm install, npm reads package-lock.json and installs the identical versions - guaranteeing that everyone has the same environment. Always commit package-lock.json to Git. It is your project's fingerprint.

08   VS Code Optimization

Supercharge Your Editor for Solidity Productivity

Essential Settings for Solidity

Open VS Code's settings (Ctrl+, or Cmd+,) and search for the following. Click "Edit in settings.json" and add:

{
  // Solidity: enable compiler warnings
  "solidity.compileUsingRemoteVersion": "0.8.24",
  "solidity.defaultCompiler": "remote",
  // Format .sol files with Prettier
  "[solidity]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "editor.formatOnSave": true
  },
  // Format .js files with Prettier
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "editor.formatOnSave": true
  },
  // Enable linting for JavaScript
  "eslint.enable": true,
  // Word wrap for long lines
  "editor.wordWrap": "on"
}

10   Glossary

Key Terms - Sub-Module 8.1

Node.js
An open-source JavaScript runtime built on Chrome's V8 engine. Executes JavaScript code outside a browser. Essential for running Hardhat, ethers.js, and deployment scripts.
npm
Node Package Manager. A command-line tool and online registry for managing JavaScript libraries. Automatically resolves and installs dependencies.
package.json
The manifest file of a Node.js project. Contains metadata, scripts, and dependency lists. Created by npm init.
package-lock.json
Auto-generated file that locks the exact version of every dependency. Ensures deterministic installs across different machines.
devDependencies
Packages needed only for development (testing, compilation), not for production. Installed with --save-dev. Hardhat and ethers are devDependencies.
SemVer
Semantic Versioning: MAJOR.MINOR.PATCH format. Caret (^) allows compatible updates; tilde (~) allows patch updates. Critical for managing dependency risks.
VS Code
Visual Studio Code: a free, extensible code editor. The preferred editor for Solidity development due to its rich extension marketplace.
Solidity Extension (Juan Blanco)
VS Code extension that provides syntax highlighting, compilation, and error diagnostics for .sol files. The most essential extension for smart contract developers.
Prettier
An opinionated code formatter that automatically formats code on save. Supports Solidity, JavaScript, and many other languages.
LTS
Long-Term Support version of Node.js. Recommended for production and development stability. Even-numbered major versions (18, 20, 22) are LTS.
