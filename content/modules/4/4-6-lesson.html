04 - Beginner-Friendly Deep Explanation

Quantum Computers - What They Actually Are

Classical vs Quantum - The Library Analogy

A classical computer is like a single person searching a library for a specific book. They go shelf by shelf, checking one book at a time. Fast? Yes - billions of "checks" per second on modern hardware. But fundamentally sequential.

A quantum computer exploits a quantum mechanical property called superposition: a quantum bit (qubit) can be 0 AND 1 simultaneously - until it is measured. This is not the same as being "50% of the time 0, 50% of the time 1." It is genuinely both at the same time. With n qubits in superposition, the quantum computer is in 2ⁿ states simultaneously - processing all of them in parallel.

Two Quantum Algorithms - Shor's and Grover's

💥

Shor's Algorithm - The Cryptography Killer

Runs on a quantum computer. Solves the Integer Factorisation Problem (breaks RSA) and the Discrete Logarithm Problem (breaks ECC, ECDSA, Diffie-Hellman) in polynomial time. Renders all current blockchain wallet cryptography obsolete. A direct existential threat.

🔍

Grover's Algorithm - The Search Speedup

Provides a quadratic speedup for searching unstructured databases. Reduces effective security of a 256-bit hash function to 128-bit equivalent. AES-128 → effectively 64-bit. AES-256 → effectively 128-bit. Fix: double all key sizes. SHA-256 hashes survive; AES-128 does not.

The Critical Insight for Blockchain

Hash functions (SHA-256, Keccak-256) and symmetric encryption (AES-256) survive the quantum era with a key-size upgrade. But all asymmetric cryptography based on factoring or discrete logs - RSA, ECC, ECDSA, EdDSA - is completely broken by Shor's Algorithm. This includes every blockchain wallet's key pair. Post-quantum cryptography replaces asymmetric crypto with new hard problems that quantum computers cannot efficiently solve.

05 - Technical Deep Dive

NIST PQC Standards - The Four Families

NIST's post-quantum standards are based on mathematical problems believed to be hard for both classical and quantum computers. Four families of algorithms emerged from the competition:

The Four Algorithm Families

Lattice-Based - PRIMARY

Learning With Errors (LWE) & Module LWE

Based on the hardness of finding a short vector in a high-dimensional lattice - a geometric structure. Equivalent to solving a system of equations with intentional noise added. Both classical and quantum computers struggle.

NIST Standard algorithms: ML-KEM (key encapsulation), ML-DSA (signatures), FN-DSA (FALCON - compact signatures)

ML-KEM: FIPS 203 ML-DSA: FIPS 204 FN-DSA: FIPS 206
Hash-Based - CONSERVATIVE

Stateless Hash-Based Signatures (SPHINCS+)

Security relies only on the collision resistance of hash functions - the most well-understood and oldest security assumption in cryptography. Even if lattice assumptions are broken someday, hash-based signatures remain secure as long as SHA-256 is secure.

NIST Standard algorithm: SLH-DSA (SPHINCS+). Larger signatures (~8 KB) but the most conservative, trusted choice.

SLH-DSA: FIPS 205 Sig size: 8 - 50 KB No trusted setup
Code-Based - LEGACY BACKUP

Error-Correcting Code Problems (Classic McEliece)

Based on the hardness of decoding a random linear error-correcting code - a problem studied since 1978. Very well understood mathematically. The trade-off: enormous public key sizes (hundreds of KB), making it impractical for most applications.

NIST alternate: Classic McEliece (KEM only, selected as backup). Not the primary recommendation due to key size.

Public key: ~260 KB 50 year history
Isogeny-Based - BROKEN

Supersingular Isogeny Diffie-Hellman (SIDH)

Based on the hardness of finding isogenies between elliptic curves. The most compact keys of any PQC candidate. However - SIDH was broken by a classical attack in 2022 by Wouter Castryck and Thomas Decru. It is no longer considered secure.

This is a reminder: PQC candidates must be conservatively evaluated. Even post-quantum algorithms can be broken by unexpected classical attacks.

Why ML-KEM and ML-DSA Are the Primary Choices

Lattice-based algorithms (ML-KEM, ML-DSA) offer the best balance of security, performance, and key/signature sizes. ML-KEM is extremely fast - faster than RSA for key encapsulation. ML-DSA signatures (~2.4 KB) are larger than ECDSA (64 bytes) but acceptable for most applications. The "ML" prefix stands for "Module Lattice" - a structured variant of lattice problems that provides both security and efficiency. SLH-DSA (SPHINCS+) is the backup: slower and larger but relies only on hash security - the most conservative and long-trusted assumption.

The Hard Problems - Why Quantum Computers Can't Solve Them

Learning With Errors (LWE) - The Lattice Problem

Imagine you are given a set of equations: a₁·s + e₁ ≈ b₁, a₂·s + e₂ ≈ b₂, ... where s is a secret vector, aᵢ are known random values, and eᵢ are small random "errors." Find s.

Without the errors, this is trivial (Gaussian elimination). With the errors, it becomes computationally infeasible - even for quantum computers. Shor's Algorithm exploits algebraic structure in RSA/ECC. LWE has no such structure - it is a high-dimensional noisy problem that defies both classical and quantum attacks.

SLH-DSA (SPHINCS+): Even simpler - security rests entirely on finding SHA-256 hash collisions. Quantum computers can search 2¹²⁸ possibilities using Grover's - but 2¹²⁸ is still infeasible. As long as SHA-256 is secure, SLH-DSA is secure. No new mathematical assumptions needed.

07 - Step-by-Step Workflow

The Five "W" Questions Answered

❓ WHY Is Post-Quantum Cryptography Urgent - If Quantum Computers Don't Exist Yet?

Three reasons make this urgent right now:

  • Harvest Now, Decrypt Later: Adversaries are recording encrypted communications today to decrypt when quantum computers are available. Any secret with a 10+ year sensitivity window is already at risk.
  • Migration takes decades: The internet has billions of devices, libraries, protocols, and systems that use RSA and ECC. Updating all of them takes 10 - 20 years. NIST's process started in 2016. The standards were finalised in 2024. Full migration will take until 2035 or later.
  • Blockchain is permanent: Every ECDSA signature ever made on Bitcoin, Ethereum, or MST is stored on-chain permanently. Even for non-sensitive data, the public keys exposed in those signatures remain quantum-vulnerable forever.

❓ WHAT Does "Quantum-Safe" Mean for a Hash Function?

SHA-256 and Keccak-256 are NOT broken by Shor's Algorithm - because hash functions don't rely on algebraic structure (factoring or discrete logs). They are affected only by Grover's Algorithm, which reduces their effective security from 256 bits to 128 bits. 128-bit security is considered more than sufficient for the foreseeable future. For comparison, breaking 128-bit security with Grover's still requires more computation than the entire global internet has performed in its history.

❓ HOW Does MST's SARAL Protocol Protect Users During Migration?

SARAL's key management runs inside a Secure Enclave (hardware chip). The key migration process for users is designed to be invisible:

1

User logs into BridgeKey Wallet as normal

Google/mobile SSO triggers enclave key derivation - unchanged from the user's perspective.

2

Enclave generates a new ML-DSA key pair alongside existing ECDSA key

The new post-quantum key pair is derived from the same secure enclave seed, ensuring the same access controls apply.

3

Hybrid transactions signed with both ECDSA and ML-DSA

During the transitional period, every transaction carries both signatures. The MST network accepts either for backward compatibility.

4

Full cutover when CRQC threat becomes real

A governance vote triggers the cutover: from block N, only ML-DSA signatures are accepted. ECDSA is deprecated. All funds previously secured by ECDSA are now on new PQ addresses - users don't notice any change in UX.

❓ WHERE Are Post-Quantum Algorithms Already Deployed?

  • Apple iMessage: PQ3 protocol (2024) - uses ML-KEM for post-quantum key exchange alongside classical ECDH. The first major consumer app to deploy PQC.
  • Signal Messenger: PQXDH - X25519 ECDH combined with ML-KEM for hybrid post-quantum key agreement (deployed 2023).
  • Cloudflare: X25519Kyber768 hybrid TLS - post-quantum key exchange on all Cloudflare connections since 2023.
  • Google Chrome: ML-KEM hybrid TLS key exchange enabled by default since Chrome 131 (2024).
  • U.S. Federal Government: NIST mandates all federal agencies migrate to PQC standards by 2030.

❓ WHAT Are the Limitations of Current PQC Algorithms?

  • ⚠️ Larger keys and signatures: ML-DSA signatures (~3.3 KB) are 50× larger than ECDSA (64 bytes). This increases transaction sizes and gas costs on-chain. FALCON (666 bytes) is a better blockchain choice.
  • ⚠️ Slower computation: Lattice operations are slower than ECC. Key generation is faster, but signing and verification take more CPU time - measurable on low-power devices.
  • ⚠️ Less mature security analysis: RSA and ECC have been analysed for 40+ years. LWE-based algorithms have 20 years of analysis - still catching up. The SIDH break in 2022 shows that PQC candidates can be surprised by classical attacks.
  • ⚠️ Stateful hash-based schemes: Some hash-based signature schemes (XMSS, LMS) are stateful - the signer must maintain and update a state counter. Losing or reusing the state breaks security. SLH-DSA (SPHINCS+) is stateless and avoids this.
10 - Real World Case Studies

PQC in Action - Three Scenarios

Case Study 1: The Bitcoin Quantum Vulnerability

Bitcoin has approximately 4 million BTC in addresses that have exposed their public keys through at least one transaction (including the legendary "Patoshi" early mined coins and Satoshi's genesis block coins). These addresses are directly vulnerable to Shor's Algorithm once a CRQC exists. The Bitcoin community is aware - but migrating Bitcoin is a governance challenge: it requires a hard fork with broad consensus, which Bitcoin famously resists.

For MST, the governance structure (on-chain DAO voting with timelocked execution) allows a coordinated, community-approved migration to PQC signatures - a significant architectural advantage over Bitcoin's informal consensus process.

Case Study 2: India's UIDAI Aadhaar - PQC Migration Mandate

Aadhaar uses ECDSA for e-Sign and RSA for certain authentication flows. UIDAI has begun evaluating NIST PQC standards for future Aadhaar infrastructure. The stakes are enormous: 1.4 billion Aadhaar records with biometric data and identity linkages. A quantum attacker who breaks Aadhaar's ECDSA keys could forge digital signatures on any document on behalf of any citizen.

DEIT (the Indian IT ministry) has begun a PQC readiness assessment for all critical national infrastructure. MST's integration with Aadhaar-based identity (via SARAL) positions it to be a reference implementation for quantum-safe digital identity in India.

Case Study 3: The 2022 SIDH Break - A Lesson in Humility

SIDH (Supersingular Isogeny Diffie-Hellman) was considered one of the most promising PQC candidates - compact keys, novel mathematics. In July 2022, just weeks before NIST was about to standardise it, two Belgian researchers (Castryck and Decru) published a classical attack that broke SIDH completely in under an hour on a laptop.

The lesson: post-quantum does not mean unbreakable. It means "we believe it is hard for quantum computers." Mathematical breakthroughs can always change that picture. This is why MST's quantum-ready pillar uses a hybrid approach (ECDSA + ML-DSA together), and why SLH-DSA (SPHINCS+) - based only on hash functions, the most conservative and best-understood assumption - is the ultimate fallback.

⚠️
Common Exam Trap - Grover's vs Shor's

Students often confuse which algorithm does what. Simple rule: Shor's breaks asymmetric crypto (RSA, ECC) - the hard problems these rely on become easy. Grover's weakens symmetric crypto and hashes - halves effective security but doesn't break them outright (just double the key size). If an exam question says "a quantum computer can break SHA-256" - that is wrong. If it says "a quantum computer can break ECDSA" - that is correct (Shor's).

12 - Security Notes

Critical Security Principles

🔴
Never Reuse Post-Quantum Addresses After Migration

Once MST migrates to post-quantum signatures, creating a new ML-DSA address and migrating funds to it is safe. However, if you then send a transaction from that new address, the ML-DSA public key is exposed on-chain. ML-DSA is quantum-safe - but this exposure is unnecessary. Best practice: use fresh addresses for high-value holdings and avoid reusing the same quantum-safe address across contexts. Defence in depth.

🔴
Hybrid Mode Is Not Weaker Than Either Algorithm Alone

A common misconception: "If ECDSA is broken by quantum computers, the hybrid (ECDSA + ML-DSA) scheme is also broken." This is wrong. In a hybrid scheme, both signatures must be valid for a transaction to be accepted. Breaking ECDSA alone is insufficient - the attacker still needs to forge the ML-DSA signature, which is quantum-safe. The hybrid is as strong as the stronger of the two algorithms.

🔬
Crypto-Agility - The Most Important PQC Design Principle

The SIDH break in 2022 proves that even NIST-evaluated PQC candidates can be broken. The most important principle for long-lived systems is crypto-agility - designing systems so that the cryptographic algorithm can be swapped out without rebuilding the entire system. MST's modular cryptographic architecture allows algorithm upgrades via governance votes and on-chain parameter changes, rather than requiring a complete protocol rewrite. This is why crypto-agility is a core design requirement for enterprise blockchain systems.

14 - Glossary

Key Terms You Must Know

Quantum Computer
A computer that exploits quantum mechanical phenomena (superposition, entanglement) to process information in fundamentally new ways. Not simply a faster classical computer - it solves specific problems with radically different complexity.
Qubit
Quantum bit. Unlike a classical bit (0 or 1), a qubit can be in superposition - simultaneously 0 and 1 until measured. n qubits in superposition represent 2ⁿ states simultaneously.
Shor's Algorithm
A quantum algorithm (1994, Peter Shor) that factors large integers and solves discrete logarithm problems in polynomial time - breaking RSA, ECC, ECDSA, EdDSA, and Diffie-Hellman.
Grover's Algorithm
A quantum algorithm (1996, Lov Grover) that searches unstructured databases with quadratic speedup. Reduces the effective security of symmetric encryption and hash functions by half. Does not break them outright.
CRQC
Cryptographically Relevant Quantum Computer. A quantum computer powerful and reliable enough to run Shor's Algorithm and break RSA-2048 or ECC-256. Does not yet exist - estimated 2030 - 2035.
Harvest Now, Decrypt Later
A threat model where adversaries record encrypted communications today and store them for decryption once a CRQC is available. Makes PQC migration urgent even before CRQCs exist.
Post-Quantum Cryptography
Cryptographic algorithms designed to be secure against both classical and quantum computers. Based on hard mathematical problems that quantum algorithms (including Shor's and Grover's) cannot efficiently solve.
Learning With Errors (LWE)
The hard mathematical problem underlying ML-KEM and ML-DSA. Given noisy linear equations, find the hidden secret vector. Believed computationally infeasible for both classical and quantum computers.
ML-KEM (FIPS 203)
Module Lattice Key Encapsulation Mechanism. The NIST-standard post-quantum replacement for RSA and ECDH key exchange. Based on Module-LWE. Fast, efficient, practical key sizes.
ML-DSA (FIPS 204)
Module Lattice Digital Signature Algorithm (CRYSTALS-Dilithium). The primary NIST-standard post-quantum replacement for ECDSA and RSA signatures. ~2.4 KB signatures. Replaces blockchain wallet signing.
SLH-DSA (FIPS 205)
Stateless Hash-based Digital Signature Algorithm (SPHINCS+). Security relies only on hash function collision resistance. Conservative, well-trusted, but large signatures (~8 KB). Quantum-safe backup.
FN-DSA / FALCON (FIPS 206)
Fast-Fourier Lattice-based Compact Signatures. Lattice-based signatures with compact output (~666 bytes) - most blockchain-practical PQC signature scheme.
Crypto-Agility
The design principle of building systems so that cryptographic algorithms can be swapped out without rebuilding the entire protocol - essential for long-lived systems navigating the PQC transition.
Hybrid Signature
A dual-signing approach that combines a classical algorithm (ECDSA) and a post-quantum algorithm (ML-DSA). Both signatures must be valid. Provides security against both classical and quantum adversaries during the transition period.

Which quantum algorithm specifically breaks RSA and ECDSA - and what mathematical problem does it solve?
  • A) Grover's Algorithm - it searches the key space to find private keys by brute force
  • B) Grover's Algorithm - it exploits the factoring structure of RSA directly
  • C) Shor's Algorithm - it solves the Integer Factorisation Problem (breaks RSA) and the Discrete Logarithm Problem (breaks ECC / ECDSA) in polynomial time
  • D) Shor's Algorithm - it reverses SHA-256 hashing to find private keys from wallet addresses
✅ Correct Answer: C

Shor's Algorithm (1994) is the quantum algorithm that makes all current asymmetric cryptography obsolete. It solves two specific hard problems in polynomial time on a quantum computer: (1) Integer Factorisation - which breaks RSA (the security of RSA relies on the difficulty of factoring n = p × q); (2) Discrete Logarithm Problem - which breaks ECC, ECDSA, EdDSA, and Diffie-Hellman (the security of these relies on the hardness of finding the scalar multiplier given two curve points). Option A describes Grover's Algorithm, which provides quadratic speedup for unstructured search but does NOT directly break asymmetric crypto. Option D is wrong - SHA-256 is not broken by Shor's (it has no algebraic structure for Shor's to exploit).

Grover's Algorithm reduces the effective security of SHA-256 from 256 bits to approximately 128 bits. What is the correct response for a system using SHA-256 that wants to maintain 256-bit post-quantum security?
  • A) Replace SHA-256 with ECDSA - which is not affected by Grover's Algorithm
  • B) Use SHA-256 twice (double-hashing) to restore the full 256-bit security level
  • C) Upgrade to SHA-512 or use SHA-256 in applications requiring only 128-bit security - 128-bit post-quantum security from SHA-256 remains adequate for most uses
  • D) Immediately switch to ML-DSA (CRYSTALS-Dilithium), as hash functions are fully broken by quantum computers
✅ Correct Answer: C

Grover's Algorithm halves the effective security of hash functions - SHA-256 provides approximately 128-bit post-quantum security. For applications that need only 128-bit security (the current standard for most uses), SHA-256 remains perfectly adequate - 128-bit security is considered sufficient for the foreseeable future. For applications requiring full 256-bit security (e.g., certain long-term archival applications), the solution is to upgrade to SHA-512 (256-bit post-quantum security with Grover's). Double-hashing (SHA-256 twice, as Bitcoin does) does NOT restore security - Grover's operates on the final hash, not the computation steps. ML-DSA is for digital signatures, not hash functions - Option D confuses the two.

The zk-STARK proof system used by StarkNet is considered post-quantum secure, while zk-SNARKs (PLONK, Groth16) are NOT. What is the fundamental cryptographic reason for this difference?
  • A) STARKs use more rounds of interaction, making them harder to attack quantumly
  • B) STARKs are based on elliptic curve cryptography which is stronger than the curves used by SNARKs
  • C) STARKs rely exclusively on hash function security (collision resistance), which is only weakened by Grover's - while SNARKs rely on elliptic curve pairings, which are broken by Shor's Algorithm
  • D) STARKs use a trusted setup, which provides additional security against quantum adversaries
✅ Correct Answer: C

This is a direct application of the quantum security analysis from this sub-module. zk-STARKs use FRI polynomial commitments, which are based entirely on hash function security. Specifically, the security reduces to finding SHA-256 collisions - which Grover's Algorithm reduces from 2²⁵⁶ to 2¹²⁸ operations, but 2¹²⁸ is still infeasible. Therefore STARKs retain adequate quantum security. zk-SNARKs (PLONK, Groth16) use KZG polynomial commitments, which rely on elliptic curve pairings - specifically the hardness of the bilinear Diffie-Hellman problem on pairing-friendly curves (BN254, BLS12-381). Shor's Algorithm solves the discrete logarithm problem on elliptic curves, breaking these pairings. A quantum computer running Shor's could forge SNARK proofs. Option D is wrong - STARKs have NO trusted setup (that's their advantage, not a quantum security mechanism).

Option A is correct for addresses that have never sent a transaction - the address is a hash of the public key, and the public key is not revealed. ✅ Correct Answer: FALSE - Full Justification Required (2 marks)

This statement is FALSE. The statement contains a fundamental logical error about how hybrid signing works.

In MST's hybrid scheme, the network requires BOTH signatures to be valid for a transaction to be accepted. This means an attacker must break BOTH ECDSA and ML-DSA to forge a transaction - not just one. This is strictly stronger than using either algorithm alone:

• If only ECDSA: a quantum attacker running Shor's can forge - one signature to break. ❌
• If only ML-DSA: a classical attacker who finds a lattice vulnerability can forge - one signature to break. ❌
• If ECDSA + ML-DSA hybrid: an attacker must simultaneously break a classical algorithm AND a post-quantum algorithm. The difficulty is multiplicative. ✅

The statement confuses "AND" logic with "OR" logic. If the network required EITHER signature to be valid (OR), then breaking one would be sufficient. But requiring BOTH (AND) means security is as strong as the stronger of the two algorithms. During the transitional period, if quantum computers arrive and break ECDSA, the ML-DSA signature still holds - funds remain secure. If ML-DSA has an unforeseen classical break, ECDSA still holds. The hybrid approach is the gold standard for cryptographic transitions. (1 mark for correct False; 1 mark for explanation of AND logic / stronger of two algorithms.)
