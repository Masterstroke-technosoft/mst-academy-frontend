02   Real-World Story

The Artist Who Went from Zero to Sold-Out Collection in One Weekend

📖 Story

Ananya, a digital illustrator from Pune, had been creating art for years and sharing it on Instagram. Her follower count was growing, but she was making almost no income from her work. A friend told her about NFTs and how artists were selling their art directly to collectors without galleries taking commissions. Ananya was intrigued but intimidated - she knew nothing about blockchain, Solidity, or crypto wallets.


She followed a guide exactly like this one. In two days, she had: generated an ERC-721 contract using the OpenZeppelin Wizard (which required no coding), uploaded her 10 artwork pieces and metadata to IPFS, deployed the contract to the MST testnet for testing, then deployed to mainnet. She built a simple minting page using the provided React template. She set a mint price of 0.01 ETH (about ₹2,500 at the time) and a maximum supply of 50 editions per artwork.


Ananya shared the minting link on her Instagram and Twitter. Within 48 hours, all 500 NFTs were minted. She earned 5 ETH (₹12.5 lakh) directly to her wallet - no gallery commission, no payment processor delay, no middleman. Her collection was verified on OpenSea, and secondary sales started generating royalties. Ananya went from "I don't know how to code" to a successful Web3 creator in one weekend. The guide that helped her is the one you're about to follow.

04   Step 1 - Generate the Smart Contract with OpenZeppelin Wizard

No Need to Write ERC-721 from Scratch

The OpenZeppelin Contract Wizard is a web-based tool that generates secure, audited smart contract code based on the options you select. It's the safest and fastest way to create an NFT contract for any EVM chain, including MST.

Open https://wizard.openzeppelin.com in your browser. Select the ERC-721 tab. You'll see a visual interface where you can toggle features on and off. Here is the configuration we'll use for our collection:

Name & Symbol: Set the name to "MST Collection" and the symbol to "MSTC".

Base URI: Leave this empty for now - we'll set it after uploading to IPFS.

Mintable: Enable this. It adds a safeMint function that only the contract owner can call.

Auto-increment IDs: Enable this. Each NFT will get a sequential token ID starting from 0.

Enumerable: Enable this. It adds functions to list all token IDs and query total supply, which marketplaces need.

URI Storage: Enable this. It adds a function to set the base URI for all token metadata.

Royalties (ERC-2981): Enable this. Set a royalty percentage (e.g., 5%) and a royalty receiver address (your wallet). This means you'll earn 5% on every secondary sale forever.

Ownable: Enable this. It restricts minting and administrative functions to the contract owner.

Pausable: Enable this (optional). It lets you pause minting in an emergency.

Click "Copy to Clipboard" and save the generated code as contracts/MyCollection.sol in your Hardhat project. The wizard produces a complete, production-ready contract with all the features you selected.

// contracts/MyCollection.sol (generated by OpenZeppelin Wizard)
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Enumerable.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721URIStorage.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Pausable.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/token/common/ERC2981.sol";

contract MyCollection is ERC721, ERC721Enumerable, ERC721URIStorage, ERC721Pausable, Ownable, ERC2981 {
    uint256 private _nextTokenId;

    constructor(address initialOwner, address royaltyReceiver)
        ERC721("MST Collection", "MSTC")
        Ownable(initialOwner)
    {
        _setDefaultRoyalty(royaltyReceiver, 500); // 5% royalty
    }

    function safeMint(address to, string memory uri) public onlyOwner {
        uint256 tokenId = _nextTokenId++;
        _safeMint(to, tokenId);
        _setTokenURI(tokenId, uri);
    }

    function pause() public onlyOwner { _pause(); }
    function unpause() public onlyOwner { _unpause(); }

    // Overrides required by Solidity
    function _update(address to, uint256 tokenId, address auth)
        internal override(ERC721, ERC721Enumerable, ERC721Pausable)
        returns (address)
    { return super._update(to, tokenId, auth); }

    function _increaseBalance(address account, uint128 value)
        internal override(ERC721, ERC721Enumerable)
    { super._increaseBalance(account, value); }

    function tokenURI(uint256 tokenId)
        public view override(ERC721, ERC721URIStorage)
        returns (string memory)
    { return super.tokenURI(tokenId); }

    function supportsInterface(bytes4 interfaceId)
        public view override(ERC721, ERC721Enumerable, ERC721URIStorage, ERC2981)
        returns (bool)
    { return super.supportsInterface(interfaceId); }
}

06   Step 3 - Prepare Metadata and Artwork

Every NFT Needs a JSON File and an Image

For a collection of 10 NFTs, create a folder metadata/ and a folder images/. For each token ID (0 through 9), you need:

1. An image file (e.g., 0.png, 1.png, etc.) in the images/ folder.

2. A JSON metadata file (e.g., 0.json, 1.json) that references the image and describes the NFT.

{
    "name": "MST Collection #0",
    "description": "The very first NFT in the MST Collection – a symbol of your journey into Web3.",
    "image": "ipfs://QmYourImagesCID/0.png",
    "attributes": [
        { "trait_type": "Background", "value": "Midnight Blue" },
        { "trait_type": "Rarity", "value": "Common" }
    ]
}

After uploading all images to IPFS, you'll get a single CID for the folder (e.g., QmYourImagesCID). Replace the image URL in every JSON file with ipfs://QmYourImagesCID/0.png (changing the filename for each). Then upload the entire metadata/ folder to IPFS, which gives you a second CID for the metadata folder.

Your base URI will be ipfs://QmYourMetadataCID/. The contract will construct the full token URI as ipfs://QmYourMetadataCID/0.json, 1.json, etc.

07   Step 4 - Upload to IPFS with Pinata

Decentralised Storage for Your Collection

Pinata is a free IPFS pinning service. Create an account at https://pinata.cloud, generate an API key, and use the Pinata web app or the Node.js SDK to upload your folders.

npm install @pinata/sdk
# upload.js
const pinataSDK = require("@pinata/sdk");
const pinata = new pinataSDK({ pinataJWTKey: "YOUR_JWT" });

async function upload() {
    const result = await pinata.pinFromFS("./images/", { pinataMetadata: { name: "MST-Collection-Images" } });
    console.log("Images CID:", result.IpfsHash);
    // Then do the same for ./metadata/
}
upload();

Once you have both CIDs, set the base URI in your contract's constructor or via a setter function. For our contract, we'll use the ERC721URIStorage approach where each token gets its own URI passed during minting. In the safeMint function, we pass ipfs://QmMetadataCID/0.json for token 0, etc. For automated minting, you can script it.

08   Step 5 - Deploy to MST Testnet

Take Your Contract Live in Minutes

// scripts/deploy.js
const hre = require("hardhat");
async function main() {
    const [deployer] = await hre.ethers.getSigners();
    console.log("Deploying with account:", deployer.address);
    const MyCollection = await hre.ethers.getContractFactory("MyCollection");
    const contract = await MyCollection.deploy(deployer.address, deployer.address);
    await contract.waitForDeployment();
    console.log("Contract deployed to:", await contract.getAddress());
}
main();

Run: npx hardhat run scripts/deploy.js --network mst_testnet

Copy the deployed contract address. Verify the contract on MSTScan using npx hardhat verify --network mst_testnet YOUR_ADDRESS with constructor arguments if needed.

10   Step 7 - List Your NFT on a Marketplace

Show the World Your Collection

After minting a few NFTs, you can list them for sale on a marketplace. For MST testnet, you can use OpenSea Testnet (if they support MST testnet) or any MST-compatible NFT marketplace. Alternatively, deploy a simple marketplace contract from the MST ecosystem, or use a generic listing platform.

To list an NFT, you typically:

1. Approve the marketplace contract to transfer your NFT by calling approve(marketplaceAddress, tokenId) on your NFT contract.

2. List the NFT on the marketplace by calling the marketplace's listItem function with the price and duration.

Take a screenshot showing your NFT listed for sale with a price. This proves you completed the full NFT lifecycle.

12   Glossary

Key Terms - Sub-Module 12.5

OpenZeppelin Wizard
A web-based tool that generates secure, audited smart contract code based on selected features. Eliminates the need to write ERC-721 from scratch.
IPFS
InterPlanetary File System - a decentralised, peer-to-peer file storage network. NFT metadata and images should be stored on IPFS for true decentralisation.
Pinata
A free IPFS pinning service that ensures your files remain available on the IPFS network. Used to upload NFT images and metadata.
ERC-2981 (Royalties)
A standard that allows NFT creators to receive a percentage of every secondary sale. Marketplaces like OpenSea automatically enforce it.
Base URI
The common prefix for all token metadata URIs. If your base URI is ipfs://Qm.../, token 0's metadata is at ipfs://Qm.../0.json.
MST Testnet
A testing network for MST Blockchain where you can deploy contracts and mint NFTs without using real value. Chain ID: 4545.
MST BridgeKey
MST's non-custodial wallet (browser extension). Used to connect to dApps, sign transactions, and manage NFTs on MST.
Marketplace Listing
The act of offering an NFT for sale on a platform like OpenSea. Requires approving the marketplace to transfer the NFT and setting a price.
