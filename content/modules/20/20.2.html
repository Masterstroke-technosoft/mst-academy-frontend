02   Real-World Introduction Story

The Developer Who Got Hired by a Tweet (And a GitHub Link)

📖 Story

In March 2023, Kunal - a self-taught blockchain developer from Lucknow - wanted to join a DeFi protocol as a smart contract engineer. He had no CS degree, no prior tech company on his resume, and no network in Bangalore's Web3 circles. All he had was a GitHub profile, a Twitter account, and a deployed contract on Polygon.


Kunal spent two weeks curating his online presence. He pinned three repositories: a gas-optimised ERC-20 token, a staking contract with 100% test coverage, and a Hardhat plugin that automated deployment scripts. Each repo had a crystal-clear README - purpose, architecture diagram, test results, and a live demo link. His Twitter bio read: “Solidity dev. I turn business logic into gas-efficient bytecode. DM for freelance.” He tweeted a thread explaining his staking contract's design choices, tagged the protocol's CTO - and the CTO replied.


The interview wasn't an interview. The CTO had already reviewed his code, checked the test coverage, and verified the on-chain contract on Polygonscan. The 30-minute call was about culture fit. Kunal got the offer - ₹18 LPA, remote - before he hung up. His portfolio did the heavy lifting before a single question was asked.

In Web3, your code, your on-chain history, and your public presence are your CV. Recruiters don't just read about your skills - they verify them. A well-crafted portfolio bridges the gap between “I can learn” and “I have already built.” This sub-module gives you the exact blueprint to replicate Kunal's result.

03   Background & History

The Death of the PDF Resume in Web3

Pre-2020
The PDF era. Resumes were static documents listing “skills: Solidity, React.” Recruiters had no way to verify claims. GitHub links were optional, often broken. On-chain activity wasn't even on the radar.
2020 - 2022
DeFi summer shifts the paradigm. Protocols started hiring based on public contributions: PRs to open-source repos, bug bounties, deployed contracts. The GitHub profile became the primary resume. On-chain transaction history served as proof of testnet experience.
2023 - 2024
The portfolio ecosystem matures. Tools like GitHub Pins, Lens Protocol, Link3, and Dune dashboards allow candidates to create dynamic, verifiable profiles. A well-crafted GitHub profile with pinned repos and a clear README is now worth more than a 2-page CV.
Today
The on-chain portfolio. A single MST contract address, verified and deployed, says more than any bullet point. Employers use block explorers to verify your work. Your CV and portfolio are inseparable from your on-chain footprint.

05   Technical Deep Dive

Building Each Component - Step by Step

We'll now construct every piece of your portfolio. Follow this blueprint exactly, and you'll have a professional, recruiter-ready presence in under a week.

A
The Web3 CV - 1 Page, No Fluff
Structure: Header (name, wallet address, GitHub, LinkedIn, MSTScan link) → 1-line summary → Technical Skills (languages, frameworks, tools) → Projects (title, what you built, tech stack, on-chain link) → Experience (if any) → Education / Certifications → Extras (hackathons, audits, blog posts).
ATS Keywords: Solidity, Hardhat, Foundry, ethers.js, Wagmi, MST, EVM, smart contracts, DeFi, ERC-20, ERC-721, gas optimisation, subgraph, React, TypeScript, Slither.
Golden Rule: Every skill listed must be demonstrable via a project on your GitHub. Never list “Solidity” without a contract to show.
B
GitHub Profile - Your Code Portfolio
Profile README: Create a special repository named exactly your username. This renders on your profile. Use it to introduce yourself, list your stack, show a visitor counter, and link to your deployed projects.
Pinned Repos (5 maximum): Pin your best work: (1) Capstone project - your MST dApp. (2) An ERC-20 or staking contract with 100% test coverage. (3) A frontend integration (React + ethers.js). (4) A Hardhat plugin or script. (5) A subgraph or data dashboard (Dune).
Each repo must have: A descriptive README (what, why, architecture, tests, deploy instructions), a clean commit history, and a link to a live demo or verified contract.
C
On-Chain Footprint - Verifiable Proof
What to include: Testnet contract addresses (MST Testnet, Sepolia), verified source code links, transaction hashes of significant interactions (deploy, staking, governance votes). If you have mainnet deployments, even better - but testnet is sufficient for entry-level.
Tool: Use MST Explorer to verify all your contracts. Include the explorer link in your CV and GitHub. Recruiters WILL click it.
Tip: Mint an NFT certificate (like your MST credential) and add its contract address to your portfolio - it proves you finished the programme.
D
Personal Website (Optional but Powerful)
Stack: Next.js + Tailwind CSS, deployed on Vercel or IPFS. Embed your on-chain activity using an API. Include a “Connect Wallet” button that displays your projects dynamically.
Content: About me, featured projects, blog (if you write), contact form.
If you don't build a site: Your GitHub profile README becomes your website. Many senior Web3 engineers use their GitHub as their primary portfolio.
E
Writing for Credibility - Blog & Twitter
Content ideas: “How I reduced gas costs by 30% in my staking contract,” “Deploying your first dApp on MST in 10 minutes,” “Why SARAL is the missing piece for Web3 mobile onboarding.”
Platforms: Medium, Hashnode, Dev.to, or your own blog. Cross-post key threads on Twitter/X.
Impact: A single well-written technical article can attract recruiters for months. It shows communication skills - critical for DevRel and senior roles.
W
WHAT
What should a Web3 CV absolutely contain?
A wallet address (or ENS), a GitHub link, a deployed contract address, and at least 3 measurable achievements (e.g., “gas-optimised a loop, reducing cost per user from 120k to 40k gas”). Generic “responsible for smart contract development” is worthless.
W
WHY
Why is GitHub more important than a CV?
Because code doesn't lie. A resume can claim “Solidity expert.” A GitHub repo with sloppy, untested code tells the truth. Hiring managers in Web3 spend 80% of their evaluation time on your repos, not your PDF.
H
HOW
How do I get noticed without experience?
Build 3 public projects that solve real problems. Deploy them on testnet. Write a blog post explaining your design decisions. Contribute to open-source MST ecosystem repos - even fixing a typo gets you on the radar. Attend hackathons and submit your project. The Web3 community rewards builders, not credentials.
L
LIMITS
What if I don't have mainnet deployments?
Testnet deployments are absolutely acceptable for entry-level roles. Ensure contracts are verified, tests are thorough, and the README explains what a mainnet version would require. A well-crafted testnet project beats a sloppy mainnet one.

09   Assessment Preparation

Top Rules - Know These Cold

01
CV length: strictly 1 page
Web3 recruiters spend 6 - 10 seconds on initial scan. If your CV is 2+ pages, it won't be read. Cut everything that isn't a project, a skill, or a verifiable achievement.
02
Every skill must have a proof
Listing “Solidity” without a deployed contract or a GitHub link is a red flag. Recruiters interpret unproven skills as “awareness” not “competence.”
03
Pinned repos are your portfolio gallery
The first thing a recruiter sees on your GitHub profile is your pinned repos. Choose them like an artist chooses paintings for an exhibition. Each one must be complete, documented, and impressive.
04
On-chain links are mandatory, not optional
A contract address on a block explorer is the Web3 equivalent of a university degree certificate. It cannot be faked. Always include it.
05
Your GitHub README is your homepage
If you do nothing else, write a compelling GitHub profile README. Many Web3 hiring managers never even open your PDF - they go straight to your GitHub and your block explorer links.

10   Glossary

Key Terms - Sub-Module 20.2

ATS (Applicant Tracking System)
Software used by HR departments to filter CVs based on keywords. To pass, your CV must include role-specific terms like “Solidity,” “Hardhat,” “MST.”
GitHub Profile README
A markdown file in a repository named after your GitHub username that renders on your profile page. Acts as your personal landing page.
Pinned Repos
Repositories you choose to showcase at the top of your GitHub profile. Limit: 6. Recruiters look here first.
On-Chain Footprint
Your verifiable activity on a blockchain: deployed contracts, transaction hashes, event logs. Proof of real development work.
Verified Contract
A smart contract whose source code has been published and matched to the deployed bytecode on a block explorer, proving authenticity.
MST Explorer
The official block explorer for the MST blockchain. Use it to verify contracts and link to specific transactions.
